# 平台运行参数与动态配置

## 能力目标

平台设置的“运行参数”用于集中控制岗位推荐、Boss 检索、Agent 执行预算和简历管理的业务边界。参数保存在 PostgreSQL `platform_setting` 的 `global/settings` 记录中，保存后必须立即作用于后续请求，不要求重启 Backend、Runtime 或 Tool；恢复默认会删除 `workspace` 覆盖项并重新应用部署时加载的默认值。

环境变量和 `application.yml` 只负责定义 Backend 启动时的默认值。平台设置加载完成后，持久化值是界面所列参数的唯一运行期有效值，不得在具体业务链路中再次直接读取同名环境变量覆盖，否则会出现界面显示与实际行为不一致。未在界面展示的工具超时、请求重试、页间延迟、Token 上限和候选总评分安全上限仍由部署配置控制。

## 参数契约

| 分组 | 参数 | 默认值 | 范围 | 实际作用点 |
| --- | --- | ---: | ---: | --- |
| 岗位推荐 | `maxJobsPerRecommend` | 15 | 1–30 | 推荐卡片每批上限、质量门达标后的提前停止数量 |
| 岗位推荐 | `recommendOverfetchFactor` | 5 | 1–10 | 候选评估池大小，和每批展示数量相乘后受 `maxJobsPerScoring` 限制 |
| 岗位推荐 | `minimumRecommendedMatchScore` | 60 | 0–100 | 简历匹配推荐质量门；候选不足时继续检索和评估，不降低门槛 |
| Boss 检索 | `bossSearchMaxPages` | 2 | 1–5 | 首轮候选预热和“换一批”补充时单次最多抓取页数 |
| Boss 检索 | `bossSearchMaxPageDepth` | 5 | 1–10 | 同一检索条件允许深入的最大 Boss 页码 |
| Boss 检索 | `bossSearchCacheTtlMinutes` | 30 | 1–1440 | 用户级候选池缓存有效期 |
| Boss 检索 | `bossSearchCooldownMinutesOnRisk` | 30 | 1–1440 | Backend 识别验证码、限流或风控信号后的暂停时长 |
| Agent 预算 | `runtimeMaxTurns` | 12 | 1–20 | Backend 发给 Runtime 的 `budget.max_turns` |
| Agent 预算 | `runtimeMaxToolCalls` | 20 | 1–30 | Backend 发给 Runtime 的 `budget.max_tool_calls` |
| Agent 预算 | `runtimeMaxFailures` | 3 | 1–10 | Backend 发给 Runtime 的 `budget.max_failures` |
| 简历管理 | `maxResumeBytes` | 5242880 | 1–20 MB | Backend 接收简历文件前的字节数校验 |
| 简历管理 | `resumeWriterVersionLimit` | 30 | 5–100 | 版本列表上限和新建版本后的历史裁剪数量 |

默认的 12 轮、20 次工具调用和 3 次连续失败与 Runtime 的 `BudgetConfig` 保持一致，避免 Backend 在请求层无意缩小 Runtime 的正式预算。岗位推荐默认展示 15 个，候选池倍率为 5，总评分上限为 80；首批候选不足或评分后未凑够展示数量时，系统会从缓存游标继续抓取尚未访问的 Boss 页并继续评分，直到展示数量、总评分上限、最大检索页深或上游结果耗尽。最低推荐分采用 60 分，仍同时要求置信度不为 low 且投递建议不是谨慎、不建议或证据不足。Boss 检索使用 2 页单次抓取、5 页最大深度、30 分钟缓存和 30 分钟遇险冷却，避免为了凑数无界访问上游。

## 加载、保存与并发可见性

Backend 初始化系统设置服务时读取持久化配置，完成字段白名单过滤、整数解析和范围归一化，再更新运行期配置快照。配置表在启动瞬间暂不可用时保留部署默认值并由服务健康轮询继续重试，不能让旁路设置故障阻断整个 Backend；设置查询、保存和恢复默认仍会返回明确的数据库错误。设置查询、保存和恢复默认都会执行同一归一化流程；保存接口返回的是实际生效值，不返回未归一化的原始输入。

运行期配置存在跨 Servlet 请求线程、SSE 工作线程和 Boss 候选池线程读取的情况，因此动态字段必须具备跨线程可见性。业务消费者每次开始新任务或新请求时读取当前值；已开始的单次任务使用启动时取得的预算和候选规模，不在执行中途改变边界，避免同一任务前后语义漂移。

## 风险边界

平台设置不能覆盖工具层和外部服务自身的安全限制。Boss CLI 的小时配额、请求抖动、单请求超时和上游冷却仍可比 Backend 设置更严格；Runtime 的服务端 Token 上限和工具权限同样保持最终约束。界面参数只能收紧或设置 Backend 发起任务时的业务预算，不能授予工具权限、绕过风控或放宽租户与用户隔离。

`maxJobsPerScoring`、`runtimeMaxTokens`、`bossSearchTargetCandidates`、`bossSearchPageDelayMillis` 和工具超时不在普通运行参数界面展示，它们是部署级安全边界。新增可编辑参数时必须同时补齐前端范围、后端白名单与归一化、真实消费点、文档和行为测试，禁止只增加表单字段。

## 验证方法

后端测试应覆盖默认值、越界归一化、保存后运行期配置立即更新、恢复默认、Runtime 请求预算、岗位候选规模与推荐阈值、Boss 分页和缓存边界、简历上传大小以及版本裁剪。前端测试应覆盖默认回退、输入范围、只提交 `workspace` 作用域和恢复默认确认。交付前执行 Backend 与 Frontend Gate，并在浏览器中修改一组非默认值后保存、刷新确认回显，再恢复默认并确认 12 个参数回到本文件定义的值。
