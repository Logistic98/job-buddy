# 题库与模拟练习整合方案

## 为什么做

现有笔试模块将题库管理、模拟考试和 LeetCode 编程练习拆成相对独立的前端入口，用户在准备求职笔试时需要在不同页面之间切换，且模拟考试只能抽取传统笔试题，无法把算法编程题与八股文选择题放到同一次限时练习中完成。与此同时，后端 interview 相关 MyBatis 映射存在接口方法名与 XML statement 不一致的问题，导致创建题目、删除题目、创建考试等接口存在运行失败风险。为了让题库管理和练习链路真正闭环，需要统一题库分类、统一练习入口、支持策略化抽题、持久化限时信息，并补齐接口验证。

## 方案是什么

题库在原有 `interview_question` 表基础上增加 `bank_type` 与 `coding_meta_json` 字段，使用 `bank_type=leetcode` 表示 LeetCode 编程题，使用 `bank_type=baguwen` 表示八股文题。原有技术分类继续保留在 `category` 字段中，例如 Java 基础、Spring、MySQL、数组与哈希表等；题型继续保留在 `question_type` 字段中，其中编程题固定为 `编程题`，八股文题支持 `单选`、`多选`、`判断` 与 `简答`。简答题按参考答案的关键片段覆盖率做轻量自动判分：参考答案按中文分号、句号、顿号、逗号和换行切分为关键片段，用户答案命中一半以上片段即判定通过。用户界面统一使用“模拟练习”概念，对外提供 `/practices` 系列接口；练习创建请求采用 rules 策略数组，每条规则增加 `bankType` 维度，使 LeetCode 编程题、八股文选择题和简答题可以按题类、分类、难度、题型和题数任意组合。

模拟练习支持“带答案”和“不带答案”两种模式。创建练习时通过请求体 `showAnswer` 字段声明，后端将其与 rules、durationMinutes 一并持久化到 `interview_exam.strategy_json`，练习详情和练习记录列表均回传 strategy。前端在带答案模式下于作答区内直接展示“参考答案（练习模式）”区块，便于边练边学；不带答案模式与真实考试一致，提交后才能看到参考答案与判分结果。练习记录侧栏对带答案的练习展示“带答案”标记。

前端交互层面，题库管理与模拟练习的页签顺序调整为题库管理在前并作为默认页签；题库管理内部以 LeetCode 编程题、八股文题两个子页签分开维护，互不混排；创建练习从占据主页面的策略卡片改为弹窗交互，弹窗内包含练习名称、限时、答案模式单选和组合规则编辑器，创建成功后自动关闭弹窗并进入限时作答。

限时练习在 `interview_exam` 表增加 `duration_minutes`、`expires_at` 与 `strategy_json` 字段，用于持久化练习时长、截止时间和抽题策略。前端创建练习后根据后端返回的 `remainingSeconds` 启动倒计时，打开进行中的练习时也以服务端剩余时间恢复计时。LeetCode 编程题的 `coding_meta_json` 保存 JavaScript 函数名、函数签名、初始代码模板和测试用例，前端在模拟练习页面内提供代码编辑、样例运行和提交前全量测试，提交时将编程题测试是否通过作为后端评分依据；八股文选择题继续按参考答案做自动判分。

## 涉及模块和接口

后端涉及 `agent-backend` 的 interview controller、service、repository、mapper、Flyway migration 和测试用例。接口包括：`GET /api/interview/questions?bankType=leetcode|baguwen` 用于按题类查询题库；`GET /api/interview/questions/meta` 用于返回题类、分类、难度、题型元数据；`POST /api/interview/practices/random` 用于按策略创建模拟练习；`GET /api/interview/practices` 与 `GET /api/interview/practices/{practiceId}` 用于查询练习记录和详情；`POST /api/interview/practices/{practiceId}/submit` 用于提交答案与编程题运行结果。

前端涉及 `agent-frontend/src/components/WrittenExamCenter.vue`、`InterviewBank.vue`、`AppSidebar.vue`、`src/api/interview.js` 与样式文件。`WrittenExamCenter` 移除单独的“编程练习”页签，保留“模拟练习”和“题库管理”两个入口；`InterviewBank` 在题库管理中展示题类字段，在练习策略中增加题类选择，在练习答题区内对编程题渲染代码编辑器和浏览器沙箱运行结果。

## 资料导入

本次通过 Flyway 迁移 `V1_12_0__Extend_written_practice_bank.sql` 导入了基础资料，包含 4 道 LeetCode 编程题和 8 道八股文选择题。LeetCode 题覆盖 Two Sum、Valid Parentheses、Merge Intervals、Binary Tree Level Order Traversal 等常见算法题；八股文选择题覆盖 Java HashMap、JVM GC Roots、Spring 事务、MySQL 联合索引、Redis 缓存穿透、Java 线程池、Kafka offset、Redis 分布式锁等求职高频知识点。后续通过 `V1_13_0__Add_short_answer_questions.sql` 补充 4 道八股文简答种子题，覆盖 HashMap 底层原理、Spring IoC、MySQL 事务隔离级别和 Redis 持久化。两次迁移均采用固定 question_id 与 `ON CONFLICT DO NOTHING`，确保重复启动不会重复导入，也不会覆盖用户后续自行维护的题目内容。

## 接口修复

本次整改同时修复了“题库与练习接口大量不可用”的两个根因。其一是本地运行的后端为旧构建，未包含 `/api/interview/practices` 系列接口和 `/questions/meta` 接口，表现为 404 与 405，需要在代码更新后重新构建并重启后端进程。其二是公共 DTO 基类 `MapBackedDto` 因 Lombok 生成的 getter 与 `@JsonAnyGetter` 平铺序列化叠加，导致响应体中同时出现 `fields` 包装对象与平铺字段，载荷翻倍且与前端约定不一致，已通过在内部 Map 字段上增加 `@JsonIgnore` 修复，仅保留平铺字段输出，该修复对继承该基类的全部 DTO 生效。

## 风险与注意事项

编程题当前使用浏览器 Web Worker 执行 JavaScript 测试用例，适合本地练习和快速验证，不等同于后端强隔离代码沙箱。后续如果需要支持 Java、Python 等多语言在线判题，应迁移到 agent-sandbox 或独立判题服务，并在后端执行测试和评分。八股文自动判分仍然是基于参考答案的轻量规则，选择题效果较稳定，简答题或开放题需要引入人工复核或模型评估。

## 如何验证

后端通过新增 `InterviewPracticeControllerTest` 覆盖题类元数据查询、按 bankType 查询题库、混合策略创建限时练习、提交八股文答案和编程题测试结果等接口。交付前需要执行 `mvn test` 或后端 gate。前端需要执行 `npm run build`，并按项目规范启动后端和前端后使用浏览器验证：进入“模拟练习”，确认题库管理能看到 LeetCode 编程题与八股文选择题两类；创建包含两类题目的混合练习；在限时答题页运行编程题样例、选择八股文答案并提交；确认练习记录保存得分且页面无报错。
